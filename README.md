## Правила ведения репозитория

***! ВАЖНО !***

Давайте будем хранить наши данные и файлы в порядке. Ведения репозитория ведите структурированно.

Все изменения ведите на своих **отдельных** ветках - `git switch -c <Название ветки>` чтобы создать ее

**Ветки** называйте строго так, чтобы было понятно ее назначение. Например, `develop-frontend` или `feature/backend`

!! Не надо коммитить ничего в main без ведома. Согласуйте все с менеджером проекта. !!

Названия коммитов старайтесь вести исключительно на английском языке и по возможности соблюдайте Соглашение о коммитах:

## Структура репозитория

#### `/backend`

Папка для хранения серверной части приложения - PostreSQL БД и GoLang API, а так же десктопное приложение для администрирования БД

#### `/frontend`

Папка для хранения клиентской части приложения - AndroidStudio проект с мобильным приложением

#### `/testing`

Юнит-тесты и все, чем занят тестировщик - тут

#### `/materials`

Здесь хранятся данные, которые скинул Федя и которые мы получили в Колледже

#### `/docs`

Здесь будут документа - отчеты, ТЗ и прочее. То, чем заняты технические писатели и менеджер проекта
